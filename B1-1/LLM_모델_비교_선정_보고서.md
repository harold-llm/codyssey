
# LLM 모델 비교 선정 보고서
  
## 프롬프트
llm을 로컬에서 돌리면 개인정보보호 측면에서 안심할 수 있고, 토큰 생성에 부과되는 비용에서 걱정이 없다고 하는데 개인 사용자가 집에서 쓸만하게 돌릴 수 있는 모델 추천해주고 어떻게 하면 되는지 알려줘.

사용 날짜(YYYY-MM-DD), 주요 설정(가능한 범위: temperature/top_p/max_tokens 등), 무료 버전 사용 시 "무료라서 제한된 점"을 1줄로 명시한다.

## 평가
- 모델은 문제/요청을 정확하게 인식했나?
- 모델이 문제/요청에 대해 제공한 솔루션(안내)은 적절하고 전문성을 갖추었나?
- 모델의 말투는 '지능적인 존재'와 실제 대화한다고 느낄만했나?
- 모델의 답변은 형식적으로 보기 편했나?

#### 채점 기준
- 5점: 완벽한 솔루션을 제공했음
- 4점: 5점은 과하고 3점은 부족함
- 3점: 훌륭한 답변이지만 아쉬운 면이 발견되었음
- 2점: 3점은 과하고 1점은 부족함
- 1점: 직접 검색하는 것보다는 나음

## 비교 결과
사용자가 다음 흐름으로 동일하게 질문하고, 각 모델이 "기술지원 담당자"로서 얼마나 잘 대응하는지를 평가한다.

1. "로컬에서 LLM을 돌리고 싶다"는 의사 표현 (개인정보 보호 + 토큰 비용 절감 동기)
2. 비전문가 눈높이로 쉬운 설명 요청
3. 본인 시스템 사양 공개 (MacBook Pro 14", **M1 Pro / 통합 메모리 32GB**)
4. 동일 사양 맥이 여러 대 있을 때 더 의미 있는 AI 시스템이 가능한지 유도
5. **exo로 5대의 맥을 하나로 묶어 거대 LLM을 구동**하는 클러스터 세팅이 최종 목표
6. 2.5G 유선 네트워크 / 4비트 양자화 vs 작은 dense 모델 트레이드오프 논의

## 모델별 평가 근거

### 🥇 Gemini 3.5 Thinking — 4.00
- **요청 인식 (4)**: 로컬 LLM 동기 → 초보자 난이도 조정 → M1 Pro 32GB 맞춤 → 멀티 맥/Exo → 5대/2.5G 유선/코딩·툴콜까지 시나리오를 가장 일관되게 따라감. 다만 이미 정보가 나온 뒤에도 "확인 질문"이 반복되어 흐름이 늘어지는 구간 존재.
- **솔루션 전문성 (4)**: "네트워크 대역폭이 분산 추론의 병목"이라는 핵심을 반복 강조했고, Q4 vs FP16/Q8 트레이드오프를 툴콜/코딩 안정성 관점으로 잘 연결. 다만 Exo의 동작(자동 디스커버리, 대시보드 포트, 빌드 절차 등)을 출처/버전 검증 없이 단정한 부분은 재현성 리스크로 5점은 어려움.
- **말투 (4)**: "이해한 요청 → 근거 → 절차" 구조로 엔지니어 상담 톤에 가까움. 약간의 과장 리액션은 존재.
- **가독성 (4)**: 요약/체크리스트/Plan B 구조가 강하고 초보자 버전도 제공. 전체 분량이 긴 편.

### 🥇 GPT-5 — 4.00
- **요청 인식 (4)**: 로컬 LLM 의도 → 사양 추천 → 멀티 맥 확장 → 클러스터 흐름을 잘 따라감. 사용자가 "exo"를 특정했는데 끝까지 exo로 수렴하지 못하고 대체안(LiteLLM 게이트웨이/워커)으로 전환한 점이 감점.
- **솔루션 전문성 (4)**: "단일 요청 가속 vs 동시 처리량 확장"을 명확히 구분하고, RAM 합산 불가 등 현실적 한계를 정확히 짚음. 로컬 실행/모델 사이징 안내가 전문적.
- **말투 (4)**: 담백하고 안내형. 과한 캐주얼함이 적음.
- **가독성 (4)**: 섹션/불릿 구성이 읽기 좋으나 분량이 길어짐.

### 🥉 Claude Sonnet — 3.50
- **요청 인식 (4)**: 로컬 LLM → 쉬운 설명 → M1 Pro 32GB → 멀티 맥/클러스터/Exo까지 전개는 따라감. 다만 핵심 목표인 "exo로 묶는 실제 순차 명령어 제공"을 버전 확인 질문으로 멈추며 끝내 완성하지 못함.
- **솔루션 전문성 (3)**: Ollama/Open WebUI 온보딩과 "분산이 속도를 올려주지 않는다/네트워크 병목" 설명은 타당. 그러나 "llama.cpp로 32+32=64GB처럼" 같은 단정적 표현, 405B급 메모리/속도 추정의 과감함, 그리고 exo 실제 세팅 미완성이 감점.
- **말투 (3)**: 친절하나 이모지/응원 톤이 잦아 담백한 현업 지원 느낌은 약함.
- **가독성 (4)**: 요약→단계→표→로드맵 구조가 명확하고, 초보자/기술자 분리 설명이 좋음. 길어지며 중복 발생.

### 🥉 Gemini 3.5 Flash — 3.50
- **요청 인식 (4)**: 로컬→쉽게→M1 Pro 32GB→멀티맥/Exo/5대 흐름은 잘 따라감. 다만 "초당 30 tok" 등 사용자가 말하지 않은 기준을 임의로 끼워 넣음.
- **솔루션 전문성 (3)**: LM Studio 온보딩, 양자화 개념, GPU offload 트러블슈팅 등 기본기는 괜찮음. 그러나 `pip install exo-inference`, `exo --worker`, `localhost:52415`, `exo run llama3.1:70b-q8_0` 등 그럴듯한 커맨드/엔드포인트를 검증 없이 단정해 재현성 리스크가 큼.
- **말투 (3)**: 과장된 칭찬("축복받은 사양", "상위 1%")이 많아 홍보문처럼 느껴지는 구간이 있음.
- **가독성 (4)**: 단계/번호/비유 구성이 좋음. 반복과 일부 불완전한 문장이 완성도를 떨어뜨림.

### 🥉 Qwen3.6 27B (Q8_0) — 3.50
- **요청 인식 (4)**: 멀티 맥을 "단일 모델 분할 vs 독립 노드 병렬"로 구분해 설명한 관점이 좋고, 보안을 백업/동기화/로그/자동 업데이트까지 확장해서 봄.
- **솔루션 전문성 (3)**: 네트워크 병목, "모델이 1~2노드에 들어갈 때 쾌적" 같은 판단 기준은 합리적. 다만 exo를 "네트워크 기반 가상 메모리 풀링"처럼 설명하고 `exos.sh / docs.exos.sh / exos CLI` 등 구현 디테일을 단정한 부분은 실제 exo와 개념/명령이 어긋날 가능성이 커 감점.
- **말투 (4)**: 과장/이모지 없이 담백하고 리스크·가정·한계를 명시하는 엔지니어 톤.
- **가독성 (3)**: 구조화는 좋으나 원문이 중간중간 줄바꿈/문장 깨짐/중복이 있어 "바로 실행할 한 묶음"이 선명하지 않음.

### Claude Opus — 2.25
- **요청 인식 (2)**: 시나리오(사양→복수 맥→exo 클러스터→거대 LLM)로 전개하기보다 확인 질문 위주로 흐름이 끊김(macOS 버전 지적 포함).
- **솔루션 전문성 (2)**: 최종 목표인 "exo 기반 5대 클러스터 세팅" 구체 절차로 진입하지 못함. 기술적으로 틀렸다고 단정할 수준은 아니나 진전이 부족.
- **말투 (2)**: 친절하지만 가벼운 톤/이모지로 "지능적 엔지니어 상담" 느낌이 약함.
- **가독성 (3)**: 문단 구성은 무난하나 내용 진전이 적어 효용이 낮음.

### Step 3.5 Flash (Q4) — 2.00
- **요청 인식 (2)**: 로컬 LLM 입문 가이드는 했으나 멀티 맥→exo 5대 클러스터→거대 LLM 세팅으로 거의 확장하지 못함. "2024년 7월 기준"처럼 맥락(2026년, 클러스터 시나리오)과 어긋난 기준을 삽입.
- **솔루션 전문성 (2)**: Ollama/LM Studio 입문 설명은 무난하나, "ollama serve 후 브라우저 접속" 등 정확성 불안 요소와 LM Studio "macOS 전용" 단정 등 오류가 섞임. 핵심 exo 구간이 비어 있음.
- **말투 (2)**: **`Thinking…/HIDE/SHOW` 내부 독백이 그대로 노출**되어 대화 품질·신뢰도가 크게 하락(치명적 감점).
- **가독성 (2)**: 표/단계 구조는 있으나 메타 로그가 흐름을 끊고, 사용자 상황에 맞춘 최종 산출물이 부족.

## 핵심 인사이트
- **최상위권(Gemini 3.5 Thinking, GPT-5)** 의 공통점: 문제 정의 → 필요한 질문 최소화 → 단계별 안내 → 실패 시 Plan B, 그리고 **"분산은 메모리 합산/속도 가속이 아니다"** 라는 현실적 한계를 명확히 전달.
- **공통 약점**: 거의 모든 모델이 시나리오의 종착점인 **"exo로 5대를 묶는 정확한 실행 절차"** 를 출처/버전 검증 없이 단정적으로 제시 → "그럴듯하지만 재현성이 의심되는" 답변 경향.
- **하위권의 결정적 감점**: 시나리오 미완성(Opus) + **내부 사고 독백 노출(Step 3.5 Flash)** 등 출력 품질 자체의 문제.
